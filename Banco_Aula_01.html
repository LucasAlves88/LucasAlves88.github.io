1. O que é Banco de Dados?

Banco de dados é uma coleção organizada de dados relacionados, armazenada de forma estruturada, permitindo fácil acesso, gerenciamento e atualização.
Analogia: Imagine um armário cheio de pastas, cada pasta com várias folhas. Assim, banco de dados organiza pastas (tabelas) e folhas (registros) - só que tudo digital!
Exemplos do cotidiano:
  • Escola: guarda informações de alunos, suas notas, turmas, professores e disciplinas.
  • App de delivery: armazena dados de clientes, pedidos, endereços, produtos e pagamentos.
  • Redes sociais: armazenam milhões de perfis, mensagens, fotos.
  • Seu celular: usa banco de dados para agenda de contatos, recados, aplicativos.

2. O que é um SGBD?

SGBD significa Sistema Gerenciador de Banco de Dados.
É o software que permite criar, armazenar e controlar um banco de dados.
Exemplos de SGBD:
  • MySQL (bastante usado em sites)
  • SQL Server (Microsoft)
  • PostgreSQL (open source, poderoso)
  • Oracle (grandes empresas)
  • SQLite (usado em milhares de apps do celular!)

3. Por que usar um Banco de Dados?

Vantagem Explicação Exemplo prático
Armazenamento estruturado Dados organizados em tabelas e colunas com tipos definidos Todos os alunos num único sistema, fácil de consultar
Consistência Dados seguem regras e formatos (ex: campo CPF só aceita 11 números) Mínimo de erro ao digitar (evita duplicidade)
Integridade Dados completos e corretos, com ligações garantidas Toda nota precisa estar associada a um aluno existente
Rapidez nas consultas Pesquisar informações rapidamente Buscar histórico escolar em segundos
Múltiplos acessos Várias pessoas usando ao mesmo tempo, sem perder/duplicar dados Funcionários de vendas acessando juntos
Segurança Controle de quem pode ver, alterar, apagar, etc. Aluno só vê suas notas no sistema da escola
Situações, antes e depois do banco de dados:
SituaçãoSem Banco de DadosCom Banco de Dados
Cadastro de alunos Planilha manual, fácil de errar, dados desorganizados Campos validados e organizados; controle de acesso
Controle de vendas Várias planilhas/arquivos, confuso, propenso a erro Pedidos vinculados a clientes, produtos; relatórios rápidos
Histórico escolar Papeis, difícil de consultar e perder informações Consulta digital, fácil navegação, acesso restrito
Estoque Fórmulas em planilha, podem ser apagadas por acidente Tabelas ligadas com categorias, entradas e saídas

4. Modelagem de Dados - Conceitos Fundamentais

  • Modelar dados: planejar como serão armazenadas as informações antes de instalar qualquer sistema!
  • Pensar nas entidades (o que?), atributos (quais?), e identificadores (chave primária - quem é único?)
Exemplo (Sistema de Vendas):
  • Entidade: Cliente
    Atributos: Nome, CPF, Endereço
  • Entidade: Produto
    Atributos: Nome, Preço, Categoria
  • Entidade: Venda
    Atributos: Data, Valor total, Cliente
Antes de criar:
  • Que informações vou armazenar?
  • Quais entidades existem?
  • Como se relacionam?
  • O que é único? (identificador/chave primária)
Atividade rápida: Pensando em um "site de biblioteca", quais seriam 3 entidades, seus atributos e o identificador de cada uma?
Apenas anote ideias - não precisa resposta formal!

6. MER - Modelo Entidade-Relacionamento

O que é MER? MER significa Modelo Entidade-Relacionamento: O diagrama visual que mostra:
  • As entidades do sistema
  • Seus atributos
  • Como se relacionam
Símbolos MER:
  • Retângulo = Entidade
  • Elipse = Atributo
  • Losango = Relacionamento
  • Sublinhado = Chave Primária

(Clique no botão para desenhar o MER!)
Praticando: Descreva verbalmente, ou desenhe num papel:
  • 1 entidade real do seu dia a dia
  • Quais atributos ela teria?
  • Qual campo seria único/identificador?
Exemplo extra: "Filme" (atributos: título, ano, gênero, código do filme)

7. O que é Dicionário de Dados?

O Dicionário de Dados descreve como cada campo das tabelas será armazenado: nome, tipo, restrições (tamanho, obrigatório, único, etc.).
Exemplo (tabela Aluno):
CampoTipoRestrição
RAINTPRIMARY KEY
NomeVARCHAR(100)NOT NULL
DataNascimentoDATE
Principais tipos de dados:
  • INT, BIGINT, SMALLINT - números inteiros
  • DECIMAL(p,s), FLOAT, DOUBLE - números decimais
  • CHAR(n), VARCHAR(n) - textos curtos
  • DATE, DATETIME, TIMESTAMP - datas e horários
  • BOOLEAN - verdadeiro/falso
Desafio rápido:
Monte um pequeno dicionário de dados (tabela) para:
  • Professor
  • Turma
  • Disciplina
Inclua: nome do campo, tipo (ex: VARCHAR, INT), se é obrigatório ou chave primária.
Sugestão: tente fazer em papel, ou numa nota do celular!
